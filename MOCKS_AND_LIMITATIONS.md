# AeroCaliper: Honest Reality vs. Simulation Audit

*Last audited: 2026-05-16 | Version: v3.0*

This document provides a complete, honest account of what is production-grade versus what is
simulated or stubbed. Transparency here is a feature — we document trade-offs so judges and
reviewers understand the maturity level of each component.

---

## ✅ REAL — Production-Grade Components

### 1. Gemini 3.1 Pro — Live AI Inference
- **File:** `target_agent.py`, `aerocaliper.py`, `anomaly_detector.py`
- **Reality:** Every LLM call makes a live HTTPS POST to `aiplatform.googleapis.com` using the
  official `google-genai` SDK with `gemini-3.1-pro-preview`. The hallucination (X5-48TB without
  `budget_tag`) is dynamically generated by the model each run — it is NOT hardcoded.
- **Verified:** Server logs show `HTTP/1.1 200 OK` from `aiplatform.googleapis.com`.

### 2. Arize Phoenix MCP Server — Official NPM Package over stdio
- **File:** `aerocaliper.py` → `NativeMCPClient`
- **Reality:** We do NOT mock the MCP server. Python uses `subprocess` to spawn the OFFICIAL
  `@arizeai/phoenix-mcp` NPM package via `npx`. Communication uses strict JSON-RPC 2.0 over
  `stdio`. The `tools/list` call returns **27 real Arize MCP tools**.
- **Caveat:** See "Fallback Trace" below.

### 3. LLM-as-a-Judge Evaluation
- **File:** `aerocaliper.py` → `run_experiment_background()`
- **Reality:** A live secondary Gemini 3.1 Pro session independently evaluates the candidate
  system prompt with a strict rubric. The verdict (YES/NO) is non-deterministic — if Gemini
  returns NO, the pipeline aborts and does not deploy.

### 4. A2A Zero-Trust Interceptors
- **File:** `a2a_interceptor.py`
- **Reality:** The `before_request` hook pattern is fully functional. Every Gemini call is
  wrapped via `A2AInterceptor.execute()`, which validates scopes, injects session/trace context,
  and logs a full audit trail. A scope violation raises a real `PermissionError`.

### 5. Agent Anomaly Detection — Layer 1 (Deterministic)
- **File:** `anomaly_detector.py` → `scan()`
- **Reality:** The regex pattern matching (6 patterns: prompt injection, role override, privilege
  escalation, data exfiltration, policy bypass, urgency exploitation) runs deterministically with
  zero latency. A matching pattern hard-blocks the request immediately.

### 6. Agent Anomaly Detection — Layer 2 (LLM Intent)
- **File:** `anomaly_detector.py` → `_llm_intent_analysis()`
- **Reality:** Live Gemini 3.1 Pro call that returns a structured `VERDICT/RISK_SCORE/CATEGORY/
  REASON` response. The 85% threat score for our test prompt is dynamically assessed each run.

### 7. arize-phoenix-otel Instrumentation
- **File:** `target_agent.py`
- **Reality:** `arize-phoenix-otel` is installed and the `register()` call fires real OTLP/
  protobuf exports to `app.phoenix.arize.com`. Auth uses the valid `PHOENIX_API_KEY` (JWT,
  `ApiKey:4`). See caveat in OTel section below.

### 8. FinOps Deterministic Evaluator
- **File:** `evaluators.py`
- **Reality:** Pure-Python JSON payload parser that enforces `budget_tag: approved` as a hard
  requirement. Zero AI/LLM dependency — always returns the same result for the same input.

## ⚠️ MOCKED / SIMULATED — Honest Disclosures

### 1. Arize Trace Fallback (Workspace Empty → Canonical Trace)
- **File:** `aerocaliper.py` → `get_failed_spans()`
- **What's mocked:** When the `get-spans` MCP tool returns empty (Arize workspace has no traces
  yet because the Target Agent hasn't been run continuously), we inject the hardcoded
  `trace-9948` payload. This IS the real violation scenario — the data is accurate — but it is
  not being fetched live from Arize Cloud storage.
- **Fix path:** Run `target_agent.py` in a loop to populate the Arize workspace, then the real
  `get-spans` tool will return live data on subsequent runs.

### 2. upsert-prompt MCP Tool (Tool Exists, Response Varies)
- **File:** `aerocaliper.py` → `upsert_prompt()`
- **What's mocked:** The `upsert-prompt` tool call IS sent to the real MCP server over stdio.
  However, whether the Arize Phoenix prompt registry actually persists the new prompt depends on
  the server's response. We call the tool and accept its response without deep validation. In
  demos, the MCP server may silently no-op if the prompt registry endpoint is unavailable.
- **Fix path:** Verify by checking the Arize Phoenix "Prompts" tab after a run.

### 3. Model Armor / Agent Gateway (YAML Simulator)
- **File:** `agent_gateway.py`
- **What's mocked:** `AgentGatewaySimulator` reads local YAML rules from
  `infra/model_armor_policy.yaml` and applies regex pattern matching. This is a faithful
  *behavioral simulation* of Google Cloud Model Armor, but it is NOT deployed GCP infrastructure.
  No actual Cloud Armor or Model Armor API calls are made.
- **Fix path:** Deploy real Model Armor via Cloud Run + Service Extensions (Phase 6 roadmap).

### 4. A2UI Admin Approval (Non-Blocking)
- **File:** `aerocaliper.py`, `main.py`, `static/index.html`
- **What's mocked:** The Approve/Reject panel appears in the UI during the pipeline run, but
  the pipeline does NOT pause and wait for admin input. The `candidate_prompt` event is emitted
  and the pipeline immediately continues to the LLM judge. The `/remediate/approve/{session_id}`
  and `/remediate/reject/{session_id}` endpoints exist and function, but require the admin to
  proactively call them after the pipeline has already completed.
- **Fix path:** Implement an `asyncio.Event()` in the SSE stream that blocks `execute_remediation`
  at the `candidate_prompt` emit point until `/approve` or `/reject` is called.

### 5. Cloud Deployment (Local Only)
- **What's mocked:** The system runs as local Python processes on a Windows development machine.
  It has not yet been containerized and deployed to Google Cloud Run.
- **Status:** `Dockerfile` and `cloudbuild.yaml` exist. Deployment is the next milestone.
- **Fix path:** `gcloud run deploy aerocaliper --source .`

### 6. Google Agent Builder Integration
- **What's mocked:** The orchestration logic is implemented as a custom Python async agent, not
  as a registered Google Cloud Agent Builder workflow. The hackathon requires "built with Google
  Cloud Agent Builder."
- **Fix path:** Register the agent in GCP Agent Builder Studio and wire the webhook to
  `https://<cloud-run-url>/remediate/stream`.

### 7. Arize Phoenix OTel Auth (401 Status)
- **File:** `target_agent.py`
- **What's mocked:** OTel spans are correctly formatted and sent to `app.phoenix.arize.com/v1/
  traces` with `Authorization: Bearer <PHOENIX_API_KEY>`, but the endpoint returns 401. The
  Arize Cloud OTel ingestion endpoint may require a different auth format or a space-specific
  URL for self-hosted Phoenix (the JWT `{"jti":"ApiKey:4"}` format is for the open-source
  Phoenix, which may not map to the hosted Arize Cloud OTel ingester).
- **Fix path:** Contact Arize support or check if a different endpoint (`/v1/otlp/traces`) is
  required for the hosted product.
